# 20260510_1800_unified_dashboard
**Task**: Refactor the web dashboard for a unified planning and execution experience.
**Status**: COMPLETE
## WHAT
- Merged Hero, Actions, and Planner into a single "Live Plan" Schedule view ✓
- Integrated active task controls (timer, buttons) directly into schedule rows ✓
- Implemented Sidecar Detail View for task inspection without page reload ✓
- Relocated "Reset Day" to Settings "Danger Zone" ✓
- Cleaned up redundant template files ✓
## HOW
- **Backend (`web_frontend.py`):**
    - Added `/fragments/task-detail/<id>` to serve detail fragments.
    - Updated `_render_page` to handle the unified context and remove old sections.
- **Frontend (`index.html` & `schedule.html`):**
    - Overhauled `index.html` with a sidecar panel and modal bulk editor.
    - Redesigned `schedule.html` to conditionally render the "Hero" state for the active task.
    - Added "Add Task" inline logic.
    - Implemented `openTaskDetail`, `closeTaskDetail`, and `toggleBulkEditor` JS.
- **Settings (`settings.html`):**
    - Added "Danger Zone" section with a protected "Reset Day" form.
- **Cleanup:**
    - Deleted `hero.html`, `actions.html`, and `planner.html` as they are now integrated into `schedule.html`.
## WHY
- The existing split between planning and execution was counterintuitive. Bringing everything into a single, evolving timeline provides a clearer "source of truth" and reduces context switching.
